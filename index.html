Section 4.9: Nurse SACT Go-Ahead [Default Category: Safety Parameters and Special Instructions] [Order Mode: Outpatient] [ONLY IF CONTAINS Nurse-administered DRUGS] [OSQ: NO DEFAULT ]
  • The sole purpose of this order is to communicate to the nurses that they need to check something before they can go ahead with chemo. For instance, let's say the pharmacist was reviewing the prescription the previous day and there were some missing blood results or it's a later Day (e.g., 2, 8, etc…) and we need to check the results just before administering the drug but hasn’t required the pharmacist to review every single day of the cycle, i.e., pharmacist reviewed Day 1, everything was ok to proceed but we need to keep an eye on specific results before each day of treatment.
  • There are 2 scenarios where this is relevant and depending on the scenario, we need to choose between 2 types of OSQs: DEFAULTED or NOT DEFAULTED. DEFAULTED is used when we want the nurses to do the check by design, e.g., for the later days. NOT DEFAULTED is used for the sole convenience of the reviewing pharmacist to give them an easy way of adding this OSQ if they for some reason, e.g., some results are taking too long to come back, require the nurse to do a final check before administering the chemo.
  • OSQs
    • For any protocol that has no classification: “NURSE GO-AHEAD - SELECT ALL NO DEFAULT” OSQ
    • For any protocol that is pre-classified as nurse release, at the protocol build level: “NURSE GO-AHEAD - SELECT ALL DEFAULTED” OSQ

    • Non-dose banded Pharmacy Stock IMPs

      • All pharmacy stock IMPs have dose banding configured within the ERX. This is not suitable for some paediatrics and adolescent trial protocols. The requesting paediatric pharmacist will indicate this in the request form. Refer to the build guide for more information.

      Traditionally, chemotherapy doses were calculated based on patients' height and weight. This led to waste, as pharmacies would have to throw away any medication that was left over from vials containing fixed doses.

      To address this issue, several hospitals have adopted a system called dose banding. With dose banding, a range of doses is delivered to closely match the vial size. This reduces waste and shortens patients' wait times, as the doses are often ready-made.

      All pharmacy stock SACT ERX (apart from a few exceptions) have dose banding configured within the ERX. This is not suitable for some paediatrics and adolescent trial protocols. The requesting paediatric pharmacist will indicate this in the request form.

      From a build perspective, the dose banding is applied in the Orderable of the SOC ERX. So what we don't want is to use that ERX as it will default to the dose banding. However, since it is a pharmacy stock imp, we do need to dispense the SOC ERX. So, to make it work what we do is we have built out a couple of lines in the CT ERX Orderable that return true when the Non Dose-Banded tag is applied and it then points to the mixture ERX directly without any banding applied.

      When this situation occurs the following will be required:

      • Use clinical trial ERX
      • Add the protocol tag “Non-Dose Banded Trial” [
      • OSQ name should begin with SACT CT(PH)-IMP
      • OSQ naming prefixes
        • SACT CT(PH)-IMP IP
        • SACT CT(PH)-IMP IP(AC)
        • SACT CT(PH)-IMP TTA
        • PO SACT CT(PH)-IMP TTA
    • IXRS/IVRS (NOW DEFUNCT)

      • IXRS is a system whereby a pharmacist will login, put in the patient details and the system will tell the pharmacist exact which vial to use based on the batch number.

      If a CT IMP requires IXRS, then add 'Refer to IXRS' to Notes to pharmacy in the OSQ and add 'IXRS' as a suffix to the OSQ name
